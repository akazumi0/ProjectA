### Loading Sprites in JavaScript

```javascript
// Preload fragment sprites
const fragmentSprites = {
  common: new Image(),
  rare: new Image(),
  epic: new Image(),
  legendary: new Image()
};

fragmentSprites.common.src = '/Assets/Graphics/kenney/PNG/Meteors/meteorBrown_med1.png';
fragmentSprites.rare.src = '/Assets/Graphics/kenney/PNG/Meteors/meteorGrey_med1.png';
fragmentSprites.epic.src = '/Assets/Graphics/kenney/PNG/Meteors/meteorBlue_med1.png';
fragmentSprites.legendary.src = '/Assets/Graphics/kenney/PNG/Meteors/meteorPink_med1.png';

// Wait for all images to load
Promise.all([
  new Promise(resolve => fragmentSprites.common.onload = resolve),
  new Promise(resolve => fragmentSprites.rare.onload = resolve),
  new Promise(resolve => fragmentSprites.epic.onload = resolve),
  new Promise(resolve => fragmentSprites.legendary.onload = resolve)
]).then(() => {
  console.log('All fragment sprites loaded!');
  startGame();
});
```

## ðŸŽ® PERFORMANCE NOTES

**Sprite vs Procedural:**
- **Pros:** Better visuals, more variety, easier to iterate
- **Cons:** More memory usage, initial load time
- **Recommendation:** Use sprites for main elements, keep procedural as fallback

**Optimization Tips:**
- Use sprite atlases for better performance
- Preload critical assets on game start
- Lazy load less-used assets
- Consider using WebP format for smaller file sizes
- Implement asset caching strategy
